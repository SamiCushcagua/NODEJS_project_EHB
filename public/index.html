API REST - Blog

Documentatie van alle beschikbare endpoints in deze API.

⚠️ Belangrijk: Zorg ervoor dat je server draait op http://localhost:3000 voordat je deze commando's gebruikt.

Basis URL

http://localhost:3000

PowerShell Voorbeelden

Kopieer deze commando's in PowerShell om de API te testen:

Gebruiker aanmaken:

Invoke-RestMethod -Uri "http://localhost:3000/api/users" -Method POST -ContentType "application/json" -Body '{ "name": "Jan Jansen", "email": "jan@voorbeeld.com", "age": 25, "bio": "Webontwikkelaar" }'

Alle gebruikers ophalen:

Invoke-RestMethod -Uri "http://localhost:3000/api/users" -Method GET

Gebruiker zoeken:

Invoke-RestMethod -Uri "http://localhost:3000/api/users?search=jan" -Method GET

Gebruiker bijwerken:

Invoke-RestMethod -Uri "http://localhost:3000/api/users/688a95408535b676463ffb2f" -Method PUT -ContentType "application/json" -Body '{ "name": "Jan Jansen Jr.", "email": "jan@voorbeeld.com", "age": 25, "bio": "Senior Webontwikkelaar" }'

Gebruiker verwijderen:

Invoke-RestMethod -Uri "http://localhost:3000/api/users/ID_VAN_GEBRUIKER" -Method DELETE

Post aanmaken:

Invoke-RestMethod -Uri "http://localhost:3000/api/posts" -Method POST -ContentType "application/json" -Body '{ "title": "Mijn eerste post", "content": "Inhoud van mijn eerste post...", "author": "ID_VAN_GEBRUIKER", "category": "deportes", "tags": ["nodejs", "express"] }'

Alle posts ophalen:

Invoke-RestMethod -Uri "http://localhost:3000/api/posts" -Method GET

Posts zoeken:

Invoke-RestMethod -Uri "http://localhost:3000/api/posts?search=nodejs&category=deportes" -Method GET

Posts van specifieke auteur:

Invoke-RestMethod -Uri "http://localhost:3000/api/posts/author/ID_VAN_AUTEUR" -Method GET

Stap-voor-stap voorbeeld:

1. Maak eerst een gebruiker aan: Invoke-RestMethod -Uri "http://localhost:3000/api/users" -Method POST -ContentType "application/json" -Body '{"name":"Anna","email":"anna@email.com","age":25}' 2. Kopieer de ID uit de response (bijv. "68876bfaf0ad7a7e434a577c") 3. Maak een post aan met die ID: Invoke-RestMethod -Uri "http://localhost:3000/api/posts" -Method POST -ContentType "application/json" -Body '{"title":"Mijn post","content":"Inhoud","author":"68876bfaf0ad7a7e434a577c","category":"deportes"}' 4. Bekijk alle posts: Invoke-RestMethod -Uri "http://localhost:3000/api/posts" -Method GET

Gebruikers Endpoints

GET
/api/users

Alle gebruikers ophalen met paginering en zoeken

Query parameters: • page - Paginanummer (standaard: 1) • limit - Items per pagina (standaard: 10) • search - Zoekterm in naam en email
GET
/api/users/:id

Specifieke gebruiker ophalen op ID

POST
/api/users

Nieuwe gebruiker aanmaken

{ "name": "Jan Jansen", "email": "jan@voorbeeld.com", "age": 25, "bio": "Webontwikkelaar" }
PUT
/api/users/:id

Bestaande gebruiker bijwerken

DELETE
/api/users/:id

Gebruiker verwijderen

Posts Endpoints

GET
/api/posts

Alle posts ophalen met filters en paginering

Query parameters: • page - Paginanummer (standaard: 1) • limit - Items per pagina (standaard: 10) • search - Zoeken in titel en inhoud • category - Filteren op categorie • author - Filteren op auteur ID
GET
/api/posts/:id

Specifieke post ophalen op ID

POST
/api/posts

Nieuwe post aanmaken

{ "title": "Mijn eerste post", "content": "Inhoud van mijn eerste post...", "author": "ID_VAN_GEBRUIKER", "category": "deportes", "tags": ["nodejs", "express"] }
PUT
/api/posts/:id

Bestaande post bijwerken

DELETE
/api/posts/:id

Post verwijderen

GET
/api/posts/author/:authorId

Alle posts van een specifieke auteur ophalen

Post Categorieën

  • sports - Posts over sport
  • other - Posts over andere onderwerpen
  • entertainment - Posts over entertainment
  • technology - Posts over technologie
  • politics - Posts over politiek

Validaties

  • Verplichte velden mogen niet leeg zijn
  • Leeftijd moet een nummer zijn tussen 13 en 120
  • Email moet geldig formaat hebben
  • Naam mag geen cijfers bevatten
  • Categorieën moeten geldig zijn

PowerShell Parameters

-Uri: De URL van de API endpoint -Method: HTTP methode (GET, POST, PUT, DELETE) -ContentType: Type data (alleen nodig voor POST/PUT) -Body: JSON data (alleen nodig voor POST/PUT)
